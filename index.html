Introduction

❓What is XariahTagger?

Xariah.net is a database for F-List eicons. The problem: eicons can only be found by their exact filename. XariahTagger lets you add plain-English tags and search by description instead.

🏷️
Tag Eicons
Add descriptive tags directly on the page.
🔍
Smart Search
Exact matches first. Phrase search. Synonyms. Negatives.
📋
One-Click Copy
Click any result to copy its BBCode instantly.
🔲
Grid Sets
Multi-tile sets copy as a complete BBCode block.
🚫
Blocked Tags
Censor any content category — stays private, never exported.
🛠️
Dev Tools
Browse by tag, synonyms, linked sets, and blocked tags.
💡

Everything is stored locally in your browser. Nothing is sent to any server. Blocked tags in particular are never exported and exist only on your device.

Getting Started

⚙️Installation

Requires Tampermonkey — a free browser extension for Chrome, Firefox, Edge, and Safari.

  1. Install the Tampermonkey extension for your browser.
  2. Open the Tampermonkey dashboard and click Create a new script.
  3. Delete the default content and paste in the full XariahTagger script.
  4. Press Ctrl + S to save.
  5. Navigate to xariah.net/eicons — the gold tag bar appears at the top. A welcome message will appear on your first visit.
Confirming the version

The tag bar shows v0.0.38 in small text. If it's different after updating, check that only one copy of the script is enabled in Tampermonkey.

Favicon activation indicator

When XariahTagger is fully operational, the browser tab icon changes to the XariahTagger tag icon. If it stays as the Xariah logo, the script's shadow-root hooks are not active — this usually means Tampermonkey's Allow user scripts browser permission is not enabled. Check Tampermonkey's extension settings for your browser.

Core Feature

✏️Tagging Eicons

Hover over any eicon on Xariah — a 🏷️ button appears in the bottom-left corner. Click it to open the tag editor.

  1. Hover over an eicon and click the 🏷️ button.
  2. Type a tag and press Enter or ,. It becomes a yellow chip.
  3. Add as many tags as you like, then click Save.

A gold glow on the 🏷️ button means the eicon is already tagged. Tags are case-insensitive. If you type a tag that is a synonym slave of an existing rule, it will automatically be replaced with the master tag, with a brief purple flash to signal the substitution.

Suggestions, Copy Last, and chip colours

As you type, two kinds of chips appear above the input:

recently used ↻
library match +

Green — your 16 most recently used tags. Blue — tags from your library matching what you're typing. Clicking either adds the tag without losing your cursor focus.

A third row of purple chips also appears:

related tag ✦

Purple — tags most commonly found together with the tags you've already added on this eicon. These are based on co-occurrence across your entire library, weighted so a tag that appears with 100% of heart-tagged eicons ranks above one that merely has a larger raw count. Purple chips don't filter when you type — they stay visible at all times based on your active tags.

The ↩ Copy Last button loads all tags from your most recently saved eicon. If the current eicon already has a grid position set (for example, from the smart grid detection), Copy Last preserves the existing grid position rather than overwriting it — only the tags are copied.

Productivity Feature

☑️Batch Tagging & Selection

Hold Ctrl and click multiple eicons — including eicons inside search results, the Dev Tools tag browser, and grid composites — to select them. A floating gold bar appears at the bottom of the screen.

💡

Selected eicons glow with a gold outline. A plain click anywhere clears the selection. The floating bar also appears inside the Dev Tools tag browser when you have eicons selected, so you can tag directly from there.

Click 🏷️ Tag Selected in the floating bar. Tags you add are merged into every selected eicon — their existing individual tags are preserved. Tags already shared by all selected eicons appear pre-loaded as yellow chips.

Creating linked sets with &&

With 2+ eicons selected in the batch modal, type && and press Enter to mark them as identical artwork uploaded under different names. See Linked Sets for the full explanation.

Search

🔎Searching — Keywords, Phrases & Negatives

Multi-word keyword search

Type any words and XariahTagger scores eicons by how well their tags match. Each word is scored independently — eicons matching more words rank higher. Results are ordered by tier (exact → starts with → whole word → substring) so the most relevant results always surface first.

Quoted phrase search

Wrap any phrase in quotes to search for it as a verbatim tag. The quoted phrase appears as a ghost chip inside the search bar and is matched exactly — no fuzzy scoring, no splitting into component words.

"heart eyes"
sparkle|

In the example above, heart eyes is a ghost chip — it only matches eicons tagged exactly heart eyes. The word sparkle is a regular keyword scored by tier. The two signals are independent and both contribute to the final ranking.

Editing a ghost chip: click it to dissolve it back into quoted text in the input so you can change it. Removing: backspace on an empty input pops the last ghost chip back to text. Deleting either quote character in the input also converts it back to plain text automatically.

Top tags row

Above Best Matches, up to 5 of the most common tags among your current results appear as small chips. Clicking a single-word chip appends it to your search. Clicking a multi-word chip converts it directly into a ghost chip for an exact match. This is a fast way to refine results without typing.

Negative terms

Prefix any keyword with - to exclude eicons tagged with it.

Example

nature -animated

Returns eicons tagged nature that are not tagged animated. Chain as many as you like: nature -animated -dark

⚠️

The - exclusion only applies at the very start of a search word. Tags like sci-fi are unaffected by it. Avoid naming tags starting with a hyphen. A search of only negative terms returns nothing — at least one positive keyword or phrase is required.

Synonyms in search

If synonym rules exist (set up in Dev Tools), searching any term in a synonym group automatically includes results for all related terms. Searching happy also surfaces eicons tagged joy if they are linked as synonyms. Synonyms are bidirectional — either end finds the full group.

Advanced Feature

🔲Grid Tags & Eicon Sets

Many eicons are tiles in a larger image — a 2×3 banner made of six eicons placed side by side. XariahTagger can recognise these sets and copy the complete BBCode in one click, including [eicon]none[/eicon] spacers for empty positions.

Creating a grid tag

In the tag editor, type #rows×cols (e.g. #2x3) and press Enter. A #2x3 × chip appears to the right of the input. A small grid renders below — click the square that represents this eicon's position in the set. Repeat for every eicon in the group.

●

When any set member appears in results, the full composite preview is shown. Click it to copy all rows. Ctrl+Click a composite to add its representative eicon to your batch selection.

How eicons are grouped automatically

XariahTagger strips trailing numbers or short suffixes and groups eicons sharing the same base name. naturebg1, naturebg2, naturebg3 → base naturebg. Bases under 3 characters are never grouped. This auto-grouping is separate from linked sets — grid tags define position in a layout, while linked sets mark identical artwork.

Smart grid detection

When you open the tag editor for an eicon that belongs to a detectable set (other eicons on the page share the same name base), a Drag grid to size panel appears below the input. A small grid of cells is shown with a ⚓ anchor at the top-left. Drag the ↕ top handle to add or remove rows, and the ↔ right handle to add or remove columns. The opposite dimension auto-adjusts to fit the detected sibling count.

When you click Confirm ✓, grid positions are automatically assigned to all detected siblings in sorted order — you don't need to open each eicon individually. The current eicon's position is pre-selected in the position picker. The same auto-assignment happens if you type a #NxM grid tag manually while the eicon is part of a detectable set.

💡

Selecting multiple eicons from the same set and using 🏷️ Tag Selected also triggers grid auto-detection — if all selected eicons share a base name, positions are assigned automatically and the grid spec appears pre-filled in the batch modal.

Deduplication

🔁Linked Sets (&&)

Some eicons are identical artwork uploaded under different filenames. Left unmanaged, every version appears in search results, wasting space. The && tag marks them as a linked set so only one representative appears per result section.

Creating a linked set

  1. Ctrl+Click 2 or more eicons with identical artwork to select them.
  2. Click 🏷️ Tag Selected in the floating bar.
  3. Type && and press Enter. The set is saved.

The && chip

When you open the tag editor for an eicon in a linked set, a && chip appears to the left of the input. It cannot be removed from here. Clicking it opens Dev Tools → Duplicates and scrolls directly to that set with a gold flash.

How deduplication works in results

Within each search result section, only the alphabetically first member of a linked set that exists in your library is shown. All others are silently skipped — the same artwork occupies exactly one slot per category.

Content Control

🚫Blocked Tags

Any tag can be blocked. Eicons whose tags include a blocked tag are censored everywhere they appear — blurred, overlaid in grey, and framed with a black-and-gold hazard border. This applies to the main Xariah browse page, search results, the tag browser, and grid composites.

Blocked tags are checked against synonym rules too — blocking a master tag also censors eicons tagged with any of its slaves, and vice versa.

⚠️ Blocked tags are never exported

Blocked tags exist only in your browser and are intentionally excluded from all export files and the developer tag import. They are entirely personal.

Managing blocked tags

Open 🛠️ Dev Tools → 🚫 Blocked. Type a tag in the input and click Block Tag (or press Enter). The list below shows all active blocked tags — click ✕ Unblock to remove one. Changes take effect immediately on the page without a reload.

Productivity Feature

👁️Tag Mode

Click the red Tag Mode button to dim every already-tagged eicon, leaving only untagged ones at full visibility. As you tag and save, eicons fade out immediately — giving you a clear view of what still needs work. Hover any dimmed eicon to temporarily restore full opacity. Click Tag Mode ON to deactivate.

Data Management

💾Import & Export

The 💾 button exports your entire library as xariah_tags.txt — one file containing tags, grid positions, synonym rules, and linked sets. Keep a copy somewhere safe; localStorage can be cleared by browser maintenance or privacy tools.

The 📥 button imports a previously exported file. A dialog shows what's in the file before anything changes:

Merge

Adds what's new from the imported file into your existing data. New tags, new synonym rules, new linked sets — all added without removing anything you already have. Recommended for most cases.

Replace

Discards all existing tags, grids, synonyms, and linked sets and replaces them entirely. Cannot be undone without a prior backup export.

🔒

Blocked tags are never included in exports or affected by imports. They remain untouched regardless of which import option you choose.

Community Feature

📡Developer Tag Library

The 📡 button fetches a curated tag library from the script's developer on GitHub and merges it into your local library. A confirmation dialog shows exactly how many new eicons, synonym rules, and linked sets will be added — you can always cancel before anything changes.

This is always a Merge. Your own tags, blocked tags, and settings are never overwritten. The file is fetched fresh each time (cache: no-cache) so you always get the latest version.

Power Feature

🛠️Dev Tools Window

The 🛠️ button opens a full-screen overlay with four tabs. Close with ✕ or press Escape.

📂 Tag Browser

The right panel lists every tag in your library as a coloured physical tag shape. Each tag is a distinct colour based on its name — consistent across sessions. Four sort controls let you order them A→Z, Z→A, most eicons first, or fewest first. The filter input narrows the list live.

Click a tag to populate the left panel with all eicons using it, displayed as a full masonry grid including composites. Ctrl+Click eicons to add them to your batch selection — a 🏷️ Tag Selected bar appears at the top of the left panel when you have eicons selected so you can batch-tag without leaving Dev Tools.

Hovering any eicon shows its name as a tooltip. Clicking 🏷️ on an eicon opens the tag editor for it directly.

🔗 Synonyms

Synonym rules make search terms interchangeable. Define happy as the master with slaves joy and cheerful — searching any of those three words returns eicons tagged with any of them. Rules are bidirectional.

When you type a slave tag in the regular tag editor and press Enter, it is automatically replaced with the master tag and the new chip briefly flashes purple to signal the substitution.

To add a rule: type the master tag, add slaves by typing and pressing Enter, then click Add Synonym Rule. To edit a rule: click ✏️ Edit — the rule is removed from the list and its values are loaded back into the form so you can modify and re-add it, including running the Clean option again.

⚠️ Clean Database option

Checking Clean database before adding replaces every instance of the slave tags with the master tag across your entire library. A confirmation shows how many eicons are affected. Cannot be undone without a backup.

🔁 Duplicates (Linked Sets)

Lists all linked sets as collapsible rows named after the first eicon in the set. Click a row to expand it and see the eicons inside as a grid. Click 🗑 Delete Group to remove the set — the eicons and their tags are unchanged, only the deduplication relationship is removed.

When you click the && chip in any tag editor, you jump directly to this tab and the relevant group is expanded and scrolled into view with a brief gold highlight.

🚫 Blocked

Add tags to your block list here. Any eicon tagged with a blocked tag is censored everywhere — blurred, overlaid in grey, and given a hazard border. Blocking works through synonyms: blocking a master also censors eicons tagged with any of its slaves.

Type a tag and press Enter or click Block Tag. Active blocked tags are listed below with individual ✕ Unblock buttons. Changes take effect immediately.

Blocked tags are personal — they are never included in any export and cannot be received via the developer tag import.
